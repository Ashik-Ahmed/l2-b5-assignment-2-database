## 8. What is the significance of the JOIN operation, and how does it work in PostgreSQL?

**JOIN** ‡¶è‡¶ï‡¶æ‡¶ß‡¶ø‡¶ï ‡¶ü‡ßá‡¶¨‡¶ø‡¶≤ ‡¶Ø‡ßÅ‡¶ï‡ßç‡¶§ ‡¶ï‡¶∞‡ßá ‡¶°‡ßá‡¶ü‡¶æ ‡¶¨‡¶ø‡¶∂‡ßç‡¶≤‡ßá‡¶∑‡¶£‡ßá ‡¶∏‡¶π‡¶æ‡ßü‡¶§‡¶æ ‡¶ï‡¶∞‡ßá‡•§

### JOIN ‡¶ß‡¶∞‡¶® ‡¶ì ‡¶â‡¶¶‡¶æ‡¶π‡¶∞‡¶£:

#### ‚úÖ INNER JOIN




#### ‚úÖ Multiple JOIN
```sql

FROM orders o
JOIN customers c ON o.customer_id = c.id
JOIN products p ON o.product_id = p.id

```

## 10. How can you calculate aggregate functions like COUNT(), SUM(), and AVG() in PostgreSQL?

### ‚úÖ COUNT()
```sql
SELECT COUNT(*) FROM employees;
SELECT COUNT(*) FROM orders WHERE status = 'Delivered';
SELECT COUNT(DISTINCT department) FROM employees;
```

### ‚úÖ SUM()
```sql
SELECT SUM(salary) FROM employees;
SELECT SUM(amount) FROM transactions WHERE transaction_date > '2023-01-01';
SELECT SUM(quantity), SUM(price) FROM order_items;
```

### ‚úÖ AVG()
```sql
SELECT AVG(rating) FROM product_reviews;
SELECT department, AVG(salary) FROM employees GROUP BY department;
SELECT ROUND(AVG(price), 2) FROM products;
```

### ‚úÖ ‡¶è‡¶ï‡¶∏‡¶æ‡¶•‡ßá ‡¶¨‡ßç‡¶Ø‡¶¨‡¶π‡¶æ‡¶∞:
```sql
SELECT 
    COUNT(*) as total_orders,
    SUM(amount) as total_revenue,
    AVG(amount) as average_order_value
FROM orders
WHERE order_date BETWEEN '2023-01-01' AND '2023-12-31';
```

### ‚úÖ FILTER clause:
```sql
SELECT 
    COUNT(*) FILTER (WHERE status = 'Completed') as completed_orders,
    COUNT(*) FILTER (WHERE status = 'Pending') as pending_orders
FROM orders;
```
